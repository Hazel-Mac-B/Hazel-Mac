## 📖 Overview

<a href="#hazel">Hazel</a> serves Mac users who maintain organized file systems — anyone who has specific places where different file types should live and wants the arrival of new files to automatically result in correct organization rather than accumulation in Downloads or the Desktop.

The <a href="#hazel">paperless workflow use case</a> is a classic Hazel application — scanning documents and having Hazel route them to appropriate folders based on content or naming, renaming scanned files with dates and descriptors, and maintaining an organized document archive without manual sorting after each scan. Hazel watches the scan output folder and processes each file as it arrives. The <a href="#hazel">developer workflow organization</a> uses Hazel to manage project files, build outputs, and downloaded assets — moving source archives to development folders, routing build outputs to test locations, and maintaining clean working directories through rules that trigger on file type and naming patterns.

The <a href="#hazel">financial document organization</a> routes bank statements, invoices, tax documents, and financial records to organized folder structures — detecting document types from filename patterns or content, applying consistent naming, and filing documents in year and category organized folders automatically. <a href="#hazel">Mac system maintenance automation</a> uses Hazel for regular cleanup tasks — emptying the Trash on a schedule, cleaning up old downloads after a set number of days, removing duplicate screenshots from the Desktop, and maintaining general disk hygiene through rules that run in the background.</p>

<a href="#hazel">Rule templates and sharing</a> provides starting points for common Hazel workflows — importing rule sets from the Hazel community for common organization tasks rather than building every rule from scratch. <a href="#hazel">Folder hierarchy creation</a> builds nested folder structures automatically as part of rule actions — creating year and month folders for document archiving, project folders for new client files, and any organizational hierarchy that should emerge from file arrival patterns.

<a href="#hazel">Multiple folder simultaneous watching</a> monitors as many folders as needed simultaneously — Downloads, Desktop, Documents, scan output, specific project folders — each with their own rule sets running independently. <a href="#hazel">Noodlesoft development</a> from an independent Mac developer with Hazel as the primary product ensures deep macOS integration, consistent macOS compatibility updates, and a tool that receives genuine attention and ongoing development.</p>

The App Sweep feature addresses a cleanup problem that macOS's standard application removal process ignores. Dragging an application to the Trash removes the application bundle but leaves behind preference files in ~/Library/Preferences, application support files in ~/Library/Application Support, caches in ~/Library/Caches, and sometimes launch agents and other system integration files. These accumulate invisibly over the lifetime of a Mac as applications are tried and discarded. Hazel's App Sweep offers to find and remove all associated files when an application moves to Trash, providing the thorough cleanup that the native Trash mechanism doesn't perform.

## 📌 FAQ

<details>
<summary>Does Hazel affect Mac performance?</summary>

Hazel runs as a background process with low resource use. Rule evaluation triggers on file system events rather than polling, keeping CPU and battery impact minimal.

</details>

<details>
<summary>Can Hazel organize photos automatically?</summary>

Yes. Import folder watching routes photo files to appropriate locations, creates date-based folder hierarchies, and triggers photo library imports.

</details>

<details>
<summary>Does Hazel integrate with other Mac automation tools?</summary>

Yes. AppleScript and shell script integration connects Hazel to any other Mac automation tool and to custom scripts for complex processing workflows.

</details>

<details>
<summary>Where can I find Hazel rule templates?</summary>

The Noodlesoft website and the Hazel user community provide rule sets for common organization workflows that import directly into Hazel.

</details>

<details>
<summary>Is Hazel a one-time purchase?</summary>

Hazel uses a one-time purchase model. Check the Noodlesoft website for current pricing.

</details>

<details>
<summary>Can Hazel rename files with dynamic naming?</summary>

Yes. Rename actions use configurable patterns including extracted date, counter, original name elements, and text replacements for consistent file naming.

</details>
